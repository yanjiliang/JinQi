## 一、项目概述

### 1.1 背景与目标

八段锦是中国传统健身功法，动作舒缓，适合全年龄段人群。然而，缺乏专业指导时，练习者容易动作变形，导致效果打折甚至受伤。本 App 利用 iPad 前置摄像头和 Apple Vision 姿态检测技术，为用户提供实时动作纠正和科学练习报告，使用户在家也能正确练习八段锦。

### 1.2 产品目标

1. **准确检测**：实时检测用户八段锦动作，与标准姿态对比，准确率 ≥ 85%
2. **即时反馈**：动作偏差在 0.5 秒内给出文字纠正提示
3. **习惯养成**：通过打卡日历和统计数据，帮助用户建立每日练习习惯
4. **零门槛使用**：界面简洁直观，老人和儿童均可独立操作

### 1.3 目标用户

| 用户群体 | 年龄范围 | 使用场景 | 核心需求 |
|---------|---------|---------|---------|
| 中老年健身爱好者 | 50-75 岁 | 晨练、居家锻炼 | 动作指导、坚持打卡 |
| 亚健康上班族 | 25-45 岁 | 午休、下班后放松 | 快速练习、进步追踪 |
| 传统文化爱好者 | 20-60 岁 | 日常养生 | 深入学习标准动作 |
| 家长/教师 | 30-50 岁 | 带领孩子/学生练习 | 教学辅助 |

### 1.4 产品定位

完全离线、免费、无广告的养生健身工具，专注于八段锦一个功法，做到极致易用和专业准确。

## 二、功能模块定义

### 2.1 模块总览

```
养生八段锦 App
├── 首页仪表盘 (Home Dashboard)
├── 练习模式 (Practice Mode)
├── 动作库 (Movement Library)
├── 练习报告 (Practice Report)
└── 练习记录与打卡 (History & Check-in)
```

### 2.2 首页仪表盘

**目的**：提供快速入口和练习概览，激励用户坚持练习。

**功能列表**：
- **快速开始按钮**：一键进入完整八段锦练习（全程约 15-20 分钟）
- **今日状态**：显示今日是否已打卡，未打卡时显示鼓励语
- **连续打卡天数**：突出展示当前连续打卡天数（streak）
- **本周统计**：本周练习次数、总时长
- **最近练习**：最近一次练习的评分摘要
- **打卡日历（月视图）**：可视化展示本月哪些天完成了练习

**UI 要求**：
- 大字体（最小 18pt）
- 高对比度配色
- 快速开始按钮视觉突出，直径 ≥ 120pt

### 2.3 练习模式

**目的**：通过摄像头实时检测，引导用户完成标准八段锦练习。

**功能列表**：
- **预备页面**：显示当前动作名称、示意图，倒计时 3 秒开始
- **实时摄像头画面**：展示前置摄像头画面（半透明覆盖层不遮挡用户）
- **标准骨架叠加**：在摄像头画面上叠加显示标准动作的骨架线（半透明蓝色）
- **用户骨架叠加**：叠加显示检测到的用户骨架（绿色=达标，红色=偏差）
- **动作进度条**：显示当前动作已保持时长 / 要求保持时长
- **实时纠正提示**：文字提示偏差部位和纠正方向（如"左臂需再抬高"）
- **动作切换**：每个动作完成后自动切换至下一动作
- **练习完成页面**：显示本次总评分和各动作评分，进入报告

**练习流程**（8个动作，每动作左右各练习 1 遍）：
1. 双手托天理三焦
2. 左右开弓似射雕（左、右各一次）
3. 调理脾胃须单举（左、右各一次）
4. 五劳七伤往后瞧（左、右各一次）
5. 摇头摆尾去心火（左、右各一次）
6. 两手攀足固肾腰
7. 攒拳怒目增气力（左、右各一次）
8. 背后七颠百病消

### 2.4 动作库

**目的**：提供八段锦每个动作的详细教学内容，可离线查阅。

**功能列表**：
- **动作列表**：展示8个动作，每条显示序号、动作名称、难度标签
- **动作详情页**：
  - 动作名称和功效说明
  - 分步骤文字说明（每步配示意图）
  - 关键要领（重点关注的身体部位）
  - 常见错误提示
  - 呼吸配合说明
  - 练习次数建议

### 2.5 练习报告

**目的**：对单次练习进行评分和分析，帮助用户了解不足并进步。

**功能列表**：
- **总评分**：0-100分，大字显示，配等级标签（优秀/良好/及格/需加强）
- **各动作评分卡**：8个动作分别的评分和简短评语
- **最佳动作**：本次得分最高的动作
- **需改进动作**：本次得分最低的动作，附具体改进建议
- **练习时长**：本次总练习时间
- **分享/保存**：生成报告图片可保存到相册（可选）
- **再练一次**：快捷重新开始练习

### 2.6 练习记录与打卡

**目的**：展示历史练习数据，提供长期进步追踪。

**功能列表**：
- **历史列表**：按日期倒序展示所有练习记录，每条显示日期、总评分、时长
- **打卡统计**：
  - 总练习次数
  - 累计练习时长
  - 当前连续打卡天数
  - 历史最长连续打卡天数
- **趋势图**：折线图显示近30次练习的评分趋势
- **打卡日历（年视图）**：类似 GitHub contribution 图，展示全年打卡热力图
- **记录删除**：长按单条记录可删除

## 四、信息架构与页面流程

### 4.1 导航结构

```
TabBar（底部标签栏，4个标签）
├── 首页（Home）              — 默认显示
├── 动作库（Library）
├── 记录（History）
└── （练习模式通过首页/浮动按钮进入，全屏覆盖，不在标签栏）
```

### 4.2 主要页面流程

```
首页
  └─[点击快速开始]─> 练习预备页（显示第一个动作）
                        └─[倒计时结束]─> 练习中（摄像头检测）
                                           └─[8个动作全部完成]─> 练习完成页
                                                                   └─[查看详情]─> 练习报告页
                                                                   └─[再练一次]─> 练习预备页

动作库
  └─[点击动作]─> 动作详情页

记录
  └─[点击历史记录]─> 练习报告页（历史查看）
```

### 4.3 关键交互规则

- 练习过程中返回：弹出确认框（"是否放弃本次练习？"），确认后退出并不保存记录
- 练习完成：自动保存记录，显示完成页面，不可取消
- 首次启动：显示摄像头权限申请说明页，申请后进入首页
- 无摄像头权限：首页显示提示横幅，点击引导去系统设置开启

## 五、非功能需求

### 5.1 性能要求

| 指标 | 要求 |
|-----|-----|
| 姿态检测帧率 | ≥ 15 FPS（目标 30 FPS） |
| 检测延迟 | < 100ms（从捕获到显示骨架） |
| 反馈延迟 | < 500ms（从检测到偏差到显示纠正提示） |
| App 启动时间 | 冷启动 < 3 秒 |
| 内存占用 | 练习中 < 300MB |
| 存储占用 | 初始安装包 < 100MB；每条练习记录 < 50KB |

### 5.2 离线支持

- **完全离线**：所有功能在无网络环境下正常运行
- **无云端依赖**：不依赖任何外部 API 或服务
- **数据本地存储**：所有练习记录存储在设备本地（SwiftData）
- **无账户系统**：不需要注册或登录

### 5.3 隐私与权限

| 权限 | 用途 | 说明 |
|-----|-----|-----|
| 摄像头 (NSCameraUsageDescription) | 姿态检测 | 必需，无此权限无法使用练习功能 |

- **摄像头数据不离开设备**：所有视频帧仅在内存中处理，不存储、不上传
- **无数据收集**：App 不收集任何用户数据，不包含追踪 SDK
- **无第三方 SDK**：仅使用 Apple 原生框架

### 5.4 无障碍支持

- 支持 Dynamic Type（系统字体大小调节）
- 支持高对比度模式
- VoiceOver 支持（动作库和记录页面）
- 练习模式中的语音提示（可在设置中开关）
